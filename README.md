## ✨ Концепция

**Calculator** — это не просто кнопки и число на экране. Это быстрый ассистент для расчётов, который:

- принимает привычный человеческий ввод;
- показывает точный результат в чистом формате;
- объясняет ход решения, когда это полезно;
- работает быстро, без фреймворков и лишнего визуального шума.

> Формат продукта: **ввёл → получил результат → при необходимости открыл ход решения**.

## 🎯 Возможности

| Направление | Что умеет |
| --- | --- |
| **Базовая математика** | `+`, `-`, `*`, `/`, скобки, степени `^` |
| **Проценты** | `15% от 320`, `50% + 12` |
| **Корни и функции** | `√144`, `sqrt(144)`, `sin`, `cos`, `tan`, `log`, `ln`, `abs` |
| **Константы** | `π`, `pi`, `e` |
| **Уравнения** | Простые линейные уравнения с `x`, например `2x+3=7` |
| **Единицы** | `mm`, `cm`, `m`, `km`, `mg`, `g`, `kg`, `s`, `min`, `h` |
| **UX** | История, share-link, CSV-экспорт, тёмная/светлая тема |
| **PWA** | Manifest + service worker для офлайн-сценариев |

## 🧪 Примеры ввода

```text
15% от 320 + √144
```

```text
📌 Результат: 60
🧮 Ход решения:
15% от 320 = 48
√144 = sqrt(144)
Итог: 60
```

```text
10 km + 500 m
```

```text
📌 Результат: 10 500 m
```

```text
2x + 3 = 7
```

```text
📌 Результат: x = 2
```

## 🖼️ Интерфейс

Дизайн построен вокруг современного, спокойного и контрастного UI:

- glassmorphism-карточки;
- мягкие градиентные фоны;
- плавные hover/focus-состояния;
- анимация появления карточек и обновления результата;
- поддержка `prefers-reduced-motion` для пользователей, которым нужны минимальные анимации.

## 🚀 Быстрый старт

Проект не требует сборки и зависимостей.

```bash
python3 -m http.server 4173
```

Затем открой:

```text
http://127.0.0.1:4173
```

Также можно открыть `index.html` напрямую в браузере, но локальный сервер лучше подходит для проверки PWA/service worker.

## 🧠 Принципы продукта

1. **Скорость важнее украшений.** Интерфейс должен мгновенно вести к результату.
2. **Понятность важнее магии.** Пользователь должен видеть, почему получился ответ.
3. **Доверие важнее маркетинга.** Формулы, ограничения и ошибки должны быть прозрачны.
4. **Реальные сценарии важнее фич ради фич.** Проценты, единицы, история и экспорт — то, что нужно каждый день.
